# Примеры шаблонов промптов для разработки

### Для максимальной релевантности ответов от LLM промпт должен включать:
1. **Тип задачи** (разработка/тестирование/анализ).
2. **Контекст**: язык, фреймворк, путь к файлу, требуемые изменения.
3. **Конкретику**: примеры кода, ожидаемое поведение, критерии успеха.
4. **Формат вывода**: если нужны только изменения — укажите «только diff», иначе будет дано пояснение.

### Выбор техники промптинга
**Выбирайте технику под задачу**: Zero-shot — для простого, One-shot — для следования стилю, Chain-of-Thought — для сложной отладки.

**Доверяйте, но проверяйте**: ИИ агент - мощный соавтор, но финальную ответственность за код несёте вы. Всегда просматривайте изменения, особенно при работе с автоматическим одобрением команд.

**Начинайте с малого**: Начните с небольших задач (рефакторинг функции, написание теста), чтобы почувствовать, как ИИ агент взаимодействует с вашим кодом.

**Экспериментируйте**: Пробуйте разные формулировки промптов и модели ИИ, чтобы найти оптимальный для себя workflow.
